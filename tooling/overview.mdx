Asciidoctor is more than a single Ruby gem. It is the center of a broad ecosystem of converters, build-tool plugins, IDE extensions, and cross-platform implementations that bring AsciiDoc processing to virtually every programming environment.

The Ruby gem is the reference implementation. Everything else in the ecosystem either wraps it (AsciidoctorJ, Asciidoctor.js) or extends it through the standard extension API (`Asciidoctor::Extensions.register`).


## Hosted git repository rendering

GitHub, GitLab, and Codeberg render `.adoc` files as HTML automatically, using Asciidoctor. You do not need to install anything. The preview is the embedded HTML output — it will be styled by the hosting platform rather than your own stylesheet, and some features such as `include::` directives may be restricted.

<Tip>
  GitLab supports a wider feature set than GitHub for AsciiDoc preview, including `include::` directives, admonition icons, diagrams, and STEM expressions.
</Tip>

## Writing your own extensions

The Asciidoctor extension API allows you to add custom behavior at every stage of the conversion pipeline. Register an extension in Ruby using `Asciidoctor::Extensions.register`:

```ruby
require 'asciidoctor'
require 'asciidoctor/extensions'

Asciidoctor::Extensions.register do
  # Preprocessor runs before parsing
  preprocessor do
    process do |document, reader|
      # modify reader lines before parsing
      reader
    end
  end

  # BlockProcessor adds a custom block macro
  block :shout do
    on_context :paragraph
    process do |parent, reader, attrs|
      lines = reader.lines.map(&:upcase)
      create_paragraph parent, lines, attrs
    end
  end
end

Asciidoctor.convert_file 'document.adoc', safe: :safe
```

Extension types available:

| Type | Purpose |
|---|---|
| `Preprocessor` | Modify raw source lines before parsing |
| `TreeProcessor` | Walk and transform the parsed document tree |
| `Postprocessor` | Modify the converted output string |
| `BlockProcessor` | Define custom delimited blocks or paragraph styles |
| `BlockMacroProcessor` | Define custom block-level macros |
| `InlineMacroProcessor` | Define custom inline macros |
| `IncludeProcessor` | Intercept `include::` directives |
| `DocinfoProcessor` | Inject content into the document header or footer |

See the [Extensions](/extensions/overview) section for full documentation.
