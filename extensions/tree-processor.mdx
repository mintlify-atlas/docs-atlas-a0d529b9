---
title: "Tree processor"
description: "Traverse and transform the parsed AST before conversion using a TreeProcessor extension."
---

A `TreeProcessor` runs after the parser has built the complete abstract syntax tree (AST) but before conversion begins. It receives the root `Document` node and can read, modify, or replace any node in the tree. If `process` returns a `Document`, that document replaces the current one; any other return value (including `nil`) leaves the tree unchanged.

## Class API

```ruby
class Asciidoctor::Extensions::TreeProcessor < Asciidoctor::Extensions::Processor
  # Override this method in your subclass.
  #
  # document - The root Asciidoctor::Document node.
  #
  # Returns the Document (same or a replacement), or nil to leave it unchanged.
  def process(document)
    raise NotImplementedError
  end
end

# Backward-compatible alias
Asciidoctor::Extensions::Treeprocessor = Asciidoctor::Extensions::TreeProcessor
```

### Useful `Document` and `AbstractBlock` methods

| Method | Description |
|--------|-------------|
| `document.blocks` | Returns the top-level child blocks. |
| `document.find_by(context: :listing)` | Searches the entire tree for nodes matching a selector. Accepts `:context`, `:style`, `:role`, and a block for custom matching. |
| `document.attr(name, default=nil)` | Reads a document attribute. |
| `document.set_attr(name, value)` | Sets a document attribute. |
| `block.context` | Returns the node's context Symbol (e.g. `:paragraph`, `:listing`, `:section`). |
| `block.title` / `block.title=` | Gets or sets the block's title. |
| `block.source_location` | Returns cursor information when `:sourcemap` is enabled. |
| `document << block` | Appends a block as the last child of `document`. |

## Example: replacing document metadata

This example is taken directly from the Asciidoctor test suite. The processor replaces the author attributes before conversion:

```ruby
require 'asciidoctor'
require 'asciidoctor/extensions'

class ReplaceAuthorTreeProcessor < Asciidoctor::Extensions::TreeProcessor
  def process document
    document.attributes['firstname'] = 'Ghost'
    document.attributes['author']    = 'Ghost Writer'
    document
  end
end

Asciidoctor::Extensions.register do
  tree_processor ReplaceAuthorTreeProcessor
end
```

## Example: appending a generated paragraph

`create_paragraph` and other helper methods from `Processor` are available inside `process`:

```ruby
class AppendSignatureTreeProcessor < Asciidoctor::Extensions::TreeProcessor
  def process document
    sig = create_paragraph document, '_Generated by Asciidoctor._', {}
    document << sig
    nil  # returning nil leaves the original document in place
  end
end
```

## Example: finding and modifying listing blocks

`document.find_by` traverses the entire tree recursively and returns all matching nodes. This processor adds a `linenums` option to every source listing that does not already have one:

```ruby
require 'asciidoctor'
require 'asciidoctor/extensions'

class AddLineNumbersTreeProcessor < Asciidoctor::Extensions::TreeProcessor
  def process document
    document.find_by(context: :listing, style: 'source').each do |block|
      block.set_attr 'linenums', '' unless block.attr? 'linenums'
    end
    nil
  end
end

Asciidoctor::Extensions.register do
  tree_processor AddLineNumbersTreeProcessor
end
```

## Example: replacing the entire document tree

If `process` returns a different `Document` object, it replaces the one being processed:

```ruby
class ReplaceTreeTreeProcessor < Asciidoctor::Extensions::TreeProcessor
  def process document
    if document.doctitle == 'Original Document'
      Asciidoctor.load %(== Replacement Document\nReplacement Author\n\ncontent)
    else
      document
    end
  end
end
```

<Note>
  When you return `nil` from `process`, the original document continues unchanged. You only need to return the document explicitly when you want to signal replacement or for clarity.
</Note>

<Tip>
  Enable `sourcemap: true` when loading your document to have each node populated with `:source_location` information (file + line number). This is useful for processors that report errors or generate cross-references.
</Tip>
