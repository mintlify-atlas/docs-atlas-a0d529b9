---
title: "Quickstart"
description: "Convert your first AsciiDoc document to HTML in minutes using the Asciidoctor CLI or Ruby API."
---

This guide walks you from a fresh Asciidoctor installation to a working HTML conversion in minutes.

<Steps>
  <Step title="Verify your installation">
    Confirm that Asciidoctor is installed and on your `PATH`:

    ```bash
    asciidoctor --version
    ```

    You should see output similar to:

    ```text
    Asciidoctor 2.0.22 [https://asciidoctor.org]
    Runtime Environment (ruby 3.4.0p0 [x86_64-linux]) (lc:UTF-8 fs:UTF-8 in:UTF-8 ex:UTF-8)
    ```

    If the command is not found, follow the [installation guide](/install) first.

    <Tip>
      Run `asciidoctor --help` to see all available flags and options.
    </Tip>
  </Step>

  <Step title="Create a sample AsciiDoc document">
    Create a file named `document.adoc` with the following content:

    ```asciidoc document.adoc
    = My First Document
    Author Name <author@example.com>
    :toc:
    :source-highlighter: highlight.js

    == Introduction

    Asciidoctor converts *AsciiDoc* source files to HTML, DocBook, and other
    formats. AsciiDoc is designed to be readable in its raw form while still
    producing beautiful rendered output.

    == Getting started

    Here are the main things you can do with Asciidoctor:

    * Convert a file to HTML with a single command
    * Use the Ruby API to convert content programmatically
    * Extend the processor with custom converters and extensions

    == A code example

    [source,ruby]
    ----
    require 'asciidoctor'

    html = Asciidoctor.convert "Hello, *World*!", safe: :safe
    puts html
    ----

    The call above returns an HTML string you can embed directly in a page.
    ```
  </Step>

  <Step title="Convert to HTML">
    Run the following command in the same directory as `document.adoc`:

    ```bash
    asciidoctor document.adoc
    ```

    Asciidoctor writes the output to `document.html` in the same directory. Open it in a browser to see the rendered result.

    To write the output to a different directory, use the `-D` flag:

    ```bash
    asciidoctor -D output document.adoc
    ```

    To convert to DocBook 5 instead of HTML, use the `-b` flag:

    ```bash
    asciidoctor -b docbook document.adoc
    ```
  </Step>

  <Step title="Convert using the Ruby API">
    You can also convert AsciiDoc content programmatically. This is useful when integrating Asciidoctor into a Ruby application, a Rails app, or a build pipeline.

    First, require the gem:

    ```ruby
    require 'asciidoctor'
    ```

    **Convert a string to embeddable HTML**

    ```ruby
    content = '_Zen_ in the art of writing https://asciidoctor.org[AsciiDoc].'
    html = Asciidoctor.convert content, safe: :safe
    puts html
    # => <div class="paragraph"><p><em>Zen</em> in the art of writing <a href="https://asciidoctor.org">AsciiDoc</a>.</p></div>
    ```

    **Convert a string to a full HTML document**

    ```ruby
    content = '_Zen_ in the art of writing https://asciidoctor.org[AsciiDoc].'
    html = Asciidoctor.convert content, header_footer: true, safe: :safe
    puts html
    # => Full HTML document including <html>, <head>, and <body> tags
    ```

    **Convert a file**

    ```ruby
    Asciidoctor.convert_file 'document.adoc', to_file: true, safe: :safe
    # => Writes document.html to the same directory as document.adoc
    ```

    **Parse the document before converting**

    If you need access to the parsed document object before conversion:

    ```ruby
    content = '_Zen_ in the art of writing https://asciidoctor.org[AsciiDoc].'
    doc = Asciidoctor.load content, header_footer: true, safe: :safe
    puts doc.doctitle   # => nil (no title declared in this string)
    html = doc.convert
    ```

    <Note>
      When using the API, the default safe mode is `:secure`. In secure mode, features like the `include` directive are disabled. Set `safe: :server` (recommended for server deployments) or `safe: :safe` to enable them.
    </Note>
  </Step>
</Steps>
